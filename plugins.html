Computational Creatures

MuseScore Plugins

The Bestiary made computational. Each creature as an interactive tool.

01The Plugin Platform

MuseScore 4 is the primary technical platform for realizing compositional creatures as interactive tools. The MuseScore Plugin API allows creatures to function as collaborative partners within the score editor itself—proposing harmonic alternatives, enforcing constraint systems, pausing for reflection, and documenting the creative dialogue between composer and algorithm.

A creature plugin does not compose for you. It composes with you. It offers suggestions rooted in philosophical constraint systems, listens to what you write, and reflects back possibilities you might not have otherwise considered. The human composer always retains authorship, intention, and the final say.

creature-plugin.ts
// Creature Plugin: The Muwashshaḥ Engine // MuseScore 4 Plugin API interface CreaturePlugin { name: string; philosophy: string; constraints: RuleSystem; aiPartner: DialogueEngine; notation: NotationMode; listening: ReflectionPrompt; } function onNoteAdded(note: Note) { // Check against maqam constraints // Suggest alternatives if outside mode // Never overwrite — only propose } function reflectionPause() { // After 8 measures, pause for listening // "What do you hear? What wants to come next?" }

02Development Phases

The creature plugin ecosystem develops in four coordinated phases, from foundational architecture through community deployment and pedagogical research integration.

In Progress

Phase 1: Foundation

Core plugin architecture, first 3 creatures, creature schema (JSON + audio + rules).

Establishing the base infrastructure: the MuseScore 4 plugin scaffold, the creature definition format (JSON schema documenting philosophy, constraints, notation modes, and reflection prompts), and the dialogue engine interface. Implementing the three primary prototypes—The Muwashshaḥ Engine, The Dice Garden, and The Listening Mirror—as proof-of-concept plugins.

Planned

Phase 2: AI Integration

Local LLM for dialogue partner, reflection prompt system.

Integrating a music-trained local language model to power the dialogue engine, enabling creatures to respond contextually to compositional choices. Developing the reflection prompt system—a set of contemplative questions and suggestions that emerge at key moments (after harmonic transitions, at phrase boundaries, upon constraint violations) to deepen the listening practice.

Planned

Phase 3: Community Platform

Open creature authoring, sharing platform, web interface.

Building a platform for the community to author and share new creatures. A web-based interface allows composers to define creatures declaratively (name, philosophy, constraint rules, notation modes), test them against the plugin API, and publish to a registry. This phase democratizes creature creation and opens the bestiary to external contributors.

Planned

Phase 4: Pedagogy Research

Classroom testing, outcome documentation, thesis integration.

Deploying creatures in educational settings to measure creative outcomes, constraint understanding, and attentional development. Documenting student compositions, feedback loops, and the evidence of philosophical constraint becoming lived practice. These outcomes become central to the final thesis, demonstrating that creatures function not merely as tools but as epistemological frameworks that reshape how composers think.
